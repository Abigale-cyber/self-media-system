# voice-script-writer

## 触发条件

- 用户说"改成口播稿"、"做短视频版本"
- 用户说"改口播"、"改成视频文案"
- 用户说"帮我改成 60 秒的短视频"

**不触发：**
- "写文章" → wechat-article-writer
- "改标题" → content-polisher
- "发草稿箱" → wechat-studio
- 已有大纲、想直接生成口播稿 → outline-to-script

## 不做什么

- 不写公众号文章（只从文章改口播）
- 不做选题、不做内容大纲
- 不做视频剪辑建议
- 不做发布

## 执行流程

### Step 1: 读取文章

读取 `../_shared/account-mode-guide.md`、`../_shared/channel-safety-guide.md` 和 `../_shared/style-guide.md`，再读取用户指定的 article-draft.md 文件并提取：
- 标题
- 核心观点
- 主要论据/案例
- 目标读者
- 文章预设
- 适用账号

### Step 2: 确认时长

询问用户目标时长：
- **60 秒** — ~180 字，适合抖音/小红书短视频
- **90 秒** — ~270 字，适合抖音中视频
- **180 秒** — ~540 字，适合 B 站/视频号

如果用户没指定，默认 90 秒。

### Step 3: 提取核心内容

从文章中提取口播稿需要的素材：
- **1 个核心观点** — 口播只讲一件事
- **1-2 个最强案例/数据** — 不贪多，选最打动人的
- **1 个安全收口** — 观众看完可思考、可确认或可进一步了解什么

如果文章内容太多无法压缩到目标时长，先和用户确认保留哪些。

### Step 4: 写逐字稿

读取 `references/voice-script-structure.md`，按四段式结构写作：

1. **钩子**（3 秒抓人）
2. **痛点/冲突**（让观众代入）
3. **方案/观点**（核心内容）
4. **安全收口**（下一步判断或互动问题）

要求：
- 口语化，像在和朋友聊天
- 加入节奏标注（停顿/加重/放慢）
- 严格控制字数在目标范围内

### Step 5: 写封面标题

生成 3 个封面标题候选：
- 短（≤15 字）、口语化、有画面感
- 适合视频封面大字体展示

### Step 6: 输出

读取 `../_shared/intermediate-format.md` 中 voice-script.md 的格式规范，按格式输出。

输出到本地文件：`voice-script-<关键词>.md`

## Guardrails

- 口播稿只讲一件事，不堆观点
- 必须有钩子（前 3 秒抓人），不用"大家好"开头
- 必须有行动号召（观众看完知道该做什么）
- 医美主题的收口不得强导流、在线给方案或暗示固定疗效
- 字数严格控制，超出目标时长的 ±10% 需要删减
- 节奏标注必须加（至少 3 处）
- 保留原文核心观点，不自由发挥
