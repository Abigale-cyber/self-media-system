# 口播稿结构规范

## 固定结构

口播稿遵循四段式结构：

1. 钩子：前 3 秒给观看理由。
2. 痛点/冲突：让用户代入具体场景。
3. 观点/方法：交付 Emma 的专业判断。
4. 行动建议：告诉用户下一步怎么判断或面诊时问什么。

## 钩子

要求：

- 不用“大家好”“今天聊聊”开头。
- 不制造容貌焦虑。
- 不承诺项目效果。
- 用具体场景、反常识或判断问题开头。

示例：

- “眼周显老，不一定是先做项目。”
- “说无恢复期之前，先看它到底指的是哪一段恢复。”
- “面诊前，这个问题最好先问清楚。”

## 痛点/冲突

- 具体到一个用户场景。
- 用“你”而不是“大家”。
- 1-2 句话点到冲突，不展开成科普长文。

## 观点/方法

- 只讲一个核心判断。
- 用“问题 -> 原因 -> 分层判断 -> 风险提醒”的顺序。
- 最多 3 个要点，避免堆项目名。
- 涉及效果、适应人群、恢复期时，必须保留个体差异和面诊判断。

## 行动建议

- 给出面诊问题、判断清单或风险提醒。
- 不做强转化，不写“马上安排/立即变美”。
- 可以温和引导用户描述情况，但不在线诊断。

## 时长控制

| 时长 | 总字数 | 钩子 | 痛点 | 观点方法 | 行动建议 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 60秒 | 约180字 | 30字 | 30字 | 100字 | 20字 |
| 90秒 | 约270字 | 40字 | 40字 | 160字 | 30字 |
| 180秒 | 约540字 | 50字 | 70字 | 370字 | 50字 |

## 节奏标注

| 标注 | 含义 |
| --- | --- |
| `(停顿)` | 让用户消化 |
| `(加重)` | 强调关键判断 |
| `(放慢)` | 说明风险、条件或面诊问题 |
| `(微笑)` | 用于缓和表达 |

## 封面标题

- 短，适合大字展示。
- 不超过 15 字。
- 不使用绝对化和功效承诺。

示例：

- “先别急着做项目”
- “眼周显老先判断”
- “恢复期要问清楚”
