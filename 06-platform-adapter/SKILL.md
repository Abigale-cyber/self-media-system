# platform-adapter

用于把已经完成审稿和合规检查的终稿适配到不同平台。它只做发布包，不负责自动发布；公众号工作台和草稿准备交给 `wechat-studio`。

## 使用范围

使用这个 Skill 处理：

- 用户说“把这篇终稿适配成小红书、视频号、公众号发布包”
- 同一篇内容需要多平台分发
- 已有文章或口播稿，需要标题、封面、简介、标签、评论区引导
- 需要明确哪个平台放长文、哪个平台放短视频、哪个平台做图文

不要用这个 Skill 写初稿、做合规审查或调用外部平台 API。

## 前置条件

- 执行前读取 `../_shared/intermediate-format.md` 中的 `publish-pack.md`，以及 `../_shared/account-mode-guide.md`、`../_shared/channel-safety-guide.md`、`../_shared/style-guide.md` 的 Emma 表达边界。
- 默认输入已经过 `medical-aesthetic-compliance-checker`。
- 必须明确终稿所属账号模式；不明确时先请求确认，不混用医生号和人格号的发布文案。
- 如果明显存在医美高风险表达，先提醒用户做合规检查，不直接适配发布。
- 如果缺少目标平台，默认输出小红书、视频号、公众号三套。

## 平台适配原则

| 平台 | 适配重点 | 内容形态 |
| --- | --- | --- |
| 小红书 | 真实感、痛点场景、收藏价值、标题和封面抓手 | 图文笔记或短视频文案 |
| 视频号（认证医生号） | 专业判断、开头 3 秒、风险边界、温和互动 | 专业口播简介、封面标题、评论引导 |
| 视频号（非认证人格号） | 观察、共鸣、认知和留白 | 人格口播简介、封面标题、评论引导 |
| 公众号 | 结构完整、专业可信、阅读体验、标题和摘要 | 长文标题、摘要、排版提示 |

医美内容统一遵守：

- 不强化外貌焦虑。
- 不承诺项目效果。
- 不用“前后变化巨大”类强对比标题。
- CTA 用“面诊时可以重点问什么”“先了解风险和适配条件”，少用“马上安排/立即变美”。

## 与 wechat-studio 衔接

当用户确认公众号发布包后，提醒下一步可以调用 `wechat-studio` 做：

- Markdown 导入
- 公众号 HTML 预览
- 主题样式调整
- 图片选择
- 人工确认
- 可选草稿箱推送

不要把外部平台 API 的自动发布流程写入本 Skill。
