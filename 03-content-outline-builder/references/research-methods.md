# 内容大纲素材补全策略

## 核心原则

内容大纲不堆资料，只选择能支撑核心判断的素材。优先用用户确认的信息；外部搜索只是增强，不是硬依赖。

## 来源优先级

### 来源 1：用户和知识库资料

优先使用：

- 用户本轮输入
- `topic-material-pack.md`
- `topic-pool.md`
- `topic-ranking.md`
- `_shared/knowledge-base-index.md` 中列出的资料名称；只有 Emma 已提供正文片段的内容才可引用

### 来源 2：平台和权威资料

当素材不足时，可补充：

- 小红书、视频号、公众号竞品内容里的评论痛点和标题角度
- 权威科普、平台规则、行业信息
- 可验证的数据、公开报道或官方说明

### 来源 3：知识推断

当用户素材不足、无法联网或用户明确不要联网时，可以用知识推断兜底，但必须标注“推断”或“待验证”。

## 降级路径

| 场景 | 策略 |
| --- | --- |
| 用户给了充足素材 | 不联网，直接整理 |
| 素材不够且可联网 | 补齐到至少 3 条，并标注来源 |
| 素材不够且无法联网 | 非医学事实可用推断辅助组织并标注待验证；核心专业依据先请 Emma 补资料 |
| 涉及强医学判断 | 不用推断当事实，改成面诊问题或风险提醒 |

## 素材质量门槛

| 检查项 | 通过标准 |
| --- | --- |
| 相关性 | 和选题核心判断直接相关 |
| 具体性 | 有场景、用户原话、步骤、条件或反例 |
| 可验证 | 数据有来源，案例有可公开边界 |
| 合规性 | 不依赖疗效承诺、夸张对比或诊疗结论 |
| 可复用 | 能放进文章、口播或发布包 |
