# 框架选择与写作规则

## 母规则

创作是对话。写每一个字之前，先想清楚读者是谁、正在担心什么、为什么需要 Emma 的判断。

## 五类文章框架

### 判断方法类

- 适用：用户不知道自己属于哪种情况。
- 结构：场景 -> 常见误区 -> 分层判断 -> 面诊问题。
- 示例：“眼周显老，先判断是哪一类问题。”

### 风险避坑类

- 适用：项目选择、恢复期、功效说法容易误导。
- 结构：高频说法 -> 风险拆解 -> 更稳妥的判断方式。
- 示例：“听到无恢复期，先问清楚这几个边界。”

### 认知纠偏类

- 适用：平台上有流行但不准确的表达。
- 结构：反常识观点 -> 原因 -> 案例/场景 -> 行动建议。
- 示例：“贵项目不一定更适合你。”

### 案例复盘类

- 适用：用户提供可公开或可脱敏案例。
- 结构：场景 -> 冲突 -> 判断逻辑 -> 经验提炼。
- 示例：“一次典型面诊问题背后的判断顺序。”

### 清单工具类

- 适用：小白收藏、面诊准备、发布平台图文。
- 结构：使用场景 -> 清单 -> 注意事项 -> 下一步。
- 示例：“面诊前可以准备的 5 个问题。”

## 分段写作法

每个章节独立写作：

1. 先写本节核心判断。
2. 用用户场景、专业逻辑或可验证素材证明它。
3. 给出可执行的判断方法或面诊问题。
4. 过渡到下一节。

## 开头原则

开头只做一件事：让目标用户相信这篇文章会回答她正在担心的问题。

避免：

- 空泛背景铺垫。
- 容貌羞辱和焦虑刺激。
- 夸张功效承诺。

## 正文原则

- 每段不超过 5 行。
- 每句话尽量不超过 25 字。
- 术语第一次出现要解释。
- 具体场景优先于抽象判断。
- 先说观点，再说理由。
- 涉及医美项目时，保留面诊和个体差异提醒。

## 结尾原则

三个要素：

1. 1-2 句总结核心判断。
2. 给出下一步判断方法或面诊问题。
3. 如需互动，用温和问题引导，不做在线诊断。

## 文章类型预设

| 预设 | 框架 | 语调 | 配图风格 | 结尾模板 | 适用 |
| --- | --- | --- | --- | --- | --- |
| `deep-insight` | 判断方法类 | 冷静专业，克制有分量 | blueprint | 面诊问题清单 | 深度判断、认知拆解 |
| `story-time` | 案例复盘类 | 温暖真实，有生活感 | warm watercolor | 经验提炼 | 脱敏案例、门诊观察 |
| `hot-take` | 认知纠偏类 | 直接但不攻击 | editorial | 风险提醒 | 平台误区、趋势纠偏 |
| `how-to` | 清单工具类 | 清晰耐心 | infographic | 步骤总结 | 面诊准备、术后注意 |
| `quick-list` | 风险避坑类 | 简洁直给 | notion | 一个核心提醒 | 小红书/视频号转图文 |

## 去 AI 痕迹

写完全文后，做一遍反 AI 检查：

| 痕迹 | 修复方式 |
| --- | --- |
| “在当今/随着时代发展” | 删掉，直接进入用户问题 |
| 三连排比 | 只保留最有用的两个，或改成不规则节奏 |
| “不仅...而且...更...” | 拆成短句 |
| “值得注意的是/需要指出的是” | 直接说结论 |
| 升华式结尾 | 改成具体判断方法或面诊问题 |
| 段落长度完全一致 | 增加长短变化 |
| 只有客观陈述 | 加入问题、场景和 Emma 的判断 |
