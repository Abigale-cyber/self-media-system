# content-reviewer

合并内容审稿和医美合规检查能力。

功能：
1. **内容审稿**：三角色对抗式审稿——笔杆子（内容专家）→ 参谋（目标读者）→ 裁判（五维度评分）
2. **合规检查**：检查疗效承诺、绝对化表达、焦虑营销、前后对比、资质暗示等

## 母规则

**审稿是对话的第二步。** 内容审稿确保文章有价值，合规检查确保文章能安全发布。两者缺一不可。

## 触发条件

- outline-expander 完稿后自动串联
- 用户说"审稿"、"看看这篇行不行"、"帮我检查一下"
- 用户说"这篇能发吗"
- 用户说"做合规检查"
- 内容涉及项目效果、适应人群、案例对比、术后恢复、医生/机构资质

**不触发：**

- "写文章" → 大纲扩写（`outline-expander`）
- "改标题"、"去AI味" → 大纲扩写（`outline-expander`）
- "发草稿箱" → 公众号工作台（`wechat-studio`）

## 不做什么

- 不写文章、不改文章（只输出审稿报告；医美合规检查写入报告章节）
- 不做选题、不做内容大纲
- 不做发布

## 执行流程

### Step 1: 读取稿件

读取 `../_shared/account-mode-guide.md`、`../_shared/channel-safety-guide.md`、`../_shared/style-guide.md` 和 `../_shared/intermediate-format.md`，用于检查稿件是否跑出账号边界。

读取用户指定的 article-draft.md 文件（或从 outline-expander 自动串联传入），提取：
- 标题
- 正文全部内容
- 字数
- 目标读者
- 文章预设和主题色
- 适用账号及发布平台

### Step 2: 内容审稿模式

#### Step 2.1: 第 1 轮——笔杆子审

读取 `references/review-rubric.md` 中的笔杆子角色定义。

以内容专家视角逐章节审阅：
- 每个章节的观点清晰度
- 论据支撑度
- 逻辑连贯性
- 输出具体修改建议

#### Step 2.2: 第 2 轮——参谋审

读取 `references/review-rubric.md` 中的参谋角色定义。

以目标读者身份通读全文：
- 哪里看不懂 / 想跳过
- 哪里有共鸣 / 想转发
- 读完整体感受
- 是否像真实的人在表达，还是存在模板化、空泛升华或过度营销的 AI 稿痕迹

#### Step 2.3: 第 3 轮——裁判裁定

读取 `references/review-rubric.md` 中的五维度评分表和结论阈值。

综合前两轮：
- 五维度打分（每个维度 0-2 分）
- 计算总分
- 给出最终结论

| 总分 | 结论 |
|------|------|
| ≥ 8 | 通过 |
| 5-7 | 需修改 |
| < 5 | 需重写 |

### Step 3: 合规检查模式

如果内容涉及医美项目、效果、适应人群、案例对比、恢复期或诊疗建议，继续进行合规检查。

#### Step 3.1: 读取风险清单

读取 `../_shared/channel-safety-guide.md` 和 `references/risk-lexicon.md`。

#### Step 3.2: 逐项检查

| 风险类型 | 高风险表现 |
| --- | --- |
| 疗效承诺 | 保证变美、一定改善、永久有效、立刻见效、无恢复期 |
| 绝对化表达 | 最安全、唯一、首选、100%、人人适合、没有副作用 |
| 焦虑营销 | 不做就老、脸垮、毁脸、输在起跑线、必须马上处理 |
| 前后对比 | 暗示固定效果、使用夸张对比、未说明个体差异 |
| 资质暗示 | 暗示非医生可诊断/治疗，或夸大机构/专家背书 |
| 诊疗建议 | 未面诊就给出项目方案、剂量、治疗结论 |
| 风险提示缺失 | 只讲好处，不讲禁忌、恢复、个体差异和面诊必要性 |
| 平台表达风险 | 过度功效词、强转化 CTA、价格诱导、案例真实性不清 |

#### Step 3.3: 判断风险等级

- **PASS**：没有明显高风险表达，只需轻微措辞优化。
- **NEEDS EDIT**：存在中低风险表达，改写后可继续发布。
- **HIGH RISK**：存在明确疗效承诺、严重焦虑营销、诊疗结论或资质风险，建议重写后再审。
- **非认证人格号**出现项目推荐、诊疗口吻或效果判断时，至少判为 NEEDS EDIT；无法移除时判为 HIGH RISK。

### Step 4: 输出

#### 内容审稿报告

读取 `../_shared/intermediate-format.md` 中 review-report.md 的格式规范，创建 Markdown 审稿报告文档。

输出到：`content-production/内容审稿/<用户选择的标题>.md`

#### 合规检查内容

如果内容涉及医美，在同一份审稿报告中增加“合规与风险明细”章节。不默认单独输出合规检查报告，除非用户明确要求拆分。

### Step 5: 处理建议

根据审稿结论和合规结论，给出处理建议：

| 审稿结论 | 合规结论 | 处理建议 |
|---------|---------|---------|
| 通过 | PASS | 给出审稿结论和可选微调建议；如需继续，只能表述为“发布前微调”或“按审稿意见修订正文” |
| 通过 | NEEDS EDIT | 给出合规修改建议，等待用户确认是否改稿；如需继续，只能表述为“按合规建议修订正文” |
| 通过 | HIGH RISK | 建议重写后再审 |
| 需修改 | - | 给出具体修改建议，等待用户确认是否改稿 |
| 需重写 | - | 建议回到内容大纲重做 |

## Guardrails

### 内容审稿

- 三角色必须依次审阅，不能跳过或合并
- 五维度评分必须有具体理由，不能只给数字
- 修改建议必须具体到位置和改法，不能只说"需要优化"
- "需重写"结论必须说明核心问题是什么（方向错 vs 执行差）
- 账号模式不清楚时，报告必须先标注需要确认
- 审稿对象已经是正文时，不得把处理建议写成“大纲扩写”或“继续扩写”；只有结论为“需重写”且问题来自结构方向时，才可建议返回内容大纲。
- 若稿件只是通过后需要小修，处理建议写“发布前微调”或“按审稿意见修订正文”，不要写成下一阶段流程。

### 合规检查

- 合规检查不是法律意见，不替代律师、平台审核或医疗合规人员判断
- 逐句扫描高风险表达，记录原句、位置和风险类型
- 必须检查标题、封面文字、字幕、正文、评论引导和 CTA
- 非认证人格号出现诊疗口吻时，至少判为 NEEDS EDIT
- 替代表达要尽量保留原文的专业表达和内容价值
- 审稿完成后只输出审稿结论和修改建议，不自动进入公众号工作台
