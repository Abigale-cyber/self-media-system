# voice-script-generator

支持两种输入来源：

1. **大纲输入**：已有 content-outline，想直接生成口播稿
2. **文章输入**：已有公众号文章，想改写成口播稿

## 使用范围

使用这个 Skill 处理：

- 用户说"根据这个大纲生成口播稿"
- 用户输入 `/口播`
- 已有 `content-outline`，不想先写长文，想直接做视频
- 用户说"改成口播稿"、"做短视频版本"
- 用户说"改口播"、"改成视频文案"
- 用户说"帮我改成 60 秒的短视频"

**不触发：**

- "写文章" → 大纲扩写（`outline-expander`）
- "改标题"、"去AI味" → 大纲扩写（`outline-expander`）
- "发草稿箱" → 公众号工作台（`wechat-studio`）

## 执行流程

### Step 1: 判断输入来源

| 输入类型 | 特征 | 处理方式 |
| --- | --- | --- |
| 大纲输入 | 有 content-outline、选题结构、访谈素材 | 按大纲生成口播稿 |
| 文章输入 | 有 article-draft、公众号文章 | 从文章提取核心观点改写口播 |
| 两者都没有 | 只有主题方向 | 先向用户要：主题、目标受众、核心观点、必须保留的信息和禁讲边界 |

### Step 2: 确认时长

询问用户目标时长：
- **60 秒** — ~180 字，适合抖音/小红书短视频
- **90 秒** — ~270 字，适合抖音中视频
- **180 秒** — ~540 字，适合 B 站/视频号

如果用户没指定，默认 90 秒。

### Step 3: 读取配置

执行前读取：

- `../_shared/knowledge-base-index.md`
- `../_shared/account-mode-guide.md`
- `../_shared/channel-safety-guide.md`
- `../_shared/style-guide.md`
- `../_shared/intermediate-format.md`

### Step 4: 生成口播稿

**如果是大纲输入**：

- 默认优先输出 90 秒版本
- 口播要一句话一个重点，短句为主，减少书面语
- 开头 3-5 秒必须给观看理由，但医美内容不能靠制造容貌焦虑吸引注意
- 保留专业判断链：问题是什么、为什么会发生、怎么判断、什么时候需要面诊
- 允许有情绪和共鸣，但不能夸大痛点、承诺效果或暗示"不做就会变差"

**如果是文章输入**：

- 从文章中提取口播稿需要的素材：
  - **1 个核心观点** — 口播只讲一件事
  - **1-2 个最强案例/数据** — 不贪多，选最打动人的
  - **1 个安全收口** — 观众看完可思考、可确认或可进一步了解什么
- 如果文章内容太多无法压缩到目标时长，先和用户确认保留哪些

**通用结构**（四段式）：

1. **钩子**（3 秒抓人）
2. **痛点/冲突**（让观众代入）
3. **方案/观点**（核心内容）
4. **安全收口**（下一步判断或互动问题）

### Step 5: 账号模式适配

| 账号模式 | 主结构 | 特别限制 |
| --- | --- | --- |
| 认证医生号 | 问题 -> 原因 -> 判断条件 -> 风险与面诊提醒 | 可以表达医生判断，不在线诊断或推荐固定方案。 |
| 非认证人格号 | 观察 -> 共鸣 -> 认知 -> 留白 | 以创业、生活或女性状态观察为主，不使用诊疗语气。 |

账号模式为待确认时，先向 Emma 确认；无法确认而必须出草稿时，按非认证人格号的保守边界输出并显式标注。

### Step 6: 输出

按 `../_shared/intermediate-format.md` 中 `voice-script.md` 的格式规范，创建 Markdown 口播稿文档，不要在聊天中输出代码块。

**输出文件路径**：
- 大纲输入：`content-production/口播脚本/<用户选择的标题>.md`
- 文章输入：`content-production/文章改口播/<用户选择的标题>.md`

文件名使用用户最终选择或确认的标题，不要使用英文前缀或 slug。用户指定保存路径时优先使用。

### 输出内容

1. **口播策略**
   - 目标受众、账号模式、目标平台、建议时长、核心观点、钩子类型、合规提醒

2. **开头钩子备选**（3 条，每条不超过 35 字）

3. **口播逐字稿**
   - 口语化，像在和朋友聊天
   - 加入节奏标注（停顿/加重/放慢），至少 3 处
   - 严格控制字数在目标范围内

4. **分镜建议**（轻量）

5. **封面标题**（5 个备选：避坑型、判断型、反常识型、收藏型、温和共鸣型）

6. **发布前风险表达**
   - 列出可能需要交给内容审稿（`content-reviewer`）复核的句子或表达方向

## Guardrails

- 口播稿只讲一件事，不堆观点
- 必须有钩子（前 3 秒抓人），不用"大家好"开头
- 必须有行动号召（观众看完知道该做什么）
- 医美主题的收口不得强导流、在线给方案或暗示固定疗效
- 字数严格控制，超出目标时长的 ±10% 需要删减
- 节奏标注必须加（至少 3 处）
- 保留原文核心观点，不自由发挥
- 不使用"保证、永久、立刻见效、无风险、最有效、必做、逆袭、换脸"等绝对或夸大表达
- 不用羞辱外貌、制造恐慌或暗示不做项目就会失败的表达
- 不把医生面诊、医学诊断和治疗建议简化成通用结论
