# 包装优化模板

## 标题优化

### 流程

1. 读原标题，判断它是信息陈述、问题、观点还是悬念。
2. 判断问题：太平、太长、太模糊、太功效化、太焦虑。
3. 从安全标题类型中选 3 个方向重写。
4. 给用户 3 个候选，不自动替换。

### 安全标题类型

| 类型 | 示例方向 |
| --- | --- |
| 判断型 | “眼周显老，先判断是哪一类问题” |
| 避坑型 | “面诊前先问清楚这几个问题” |
| 反常识型 | “有些问题，不是越早做项目越好” |
| 清单型 | “恢复期需要确认的 5 件事” |
| 温和共鸣型 | “看起来疲惫，不一定只是熬夜” |

### 标题修复

| 问题 | 修复 |
| --- | --- |
| 太长 | 压缩到核心判断 |
| 太平 | 加具体场景或判断冲突 |
| 太模糊 | 加人群、场景或问题 |
| 太焦虑 | 改成判断方法或风险提醒 |
| 太功效化 | 删除效果承诺，改成条件和边界 |

## 开头优化

### 流程

1. 判断当前开头类型：铺垫型、问题型、场景型、直入型。
2. 判断问题：太慢、太干、没有用户场景、表达有风险。
3. 从 `../_shared/hook-and-ending.md` 里选合适模板。
4. 给 3 个候选。

### 检查标准

- 前 3 句能抓住目标用户。
- 不超过 4 句话。
- 不用空泛时代背景。
- 不制造容貌焦虑。
- 和标题承诺一致。

## 结尾优化

### 流程

1. 判断当前结尾类型：总结型、行动建议型、互动型、开放型。
2. 判断问题：太散、太升华、太强转化、缺少判断方法。
3. 给 2 个候选。

### 检查标准

- 有核心判断总结。
- 有下一步判断方法或面诊问题。
- 不写“马上安排/立即变美”。
- 和 Emma 的克制专业调性一致。

## 句子润色

- 删掉不影响理解的词。
- 长句拆短句。
- 抽象描述换成用户场景。
- 绝对化表达改成条件表达。
- 功效承诺改成可能性、适配条件和风险提醒。
- 保留 Emma 的专业判断，不改文章方向。
