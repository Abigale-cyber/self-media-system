# content-polisher

## 触发条件

- 用户说"改标题"、"标题不够好"
- 用户说"开头重写"、"开头不够吸引人"
- 用户说"去AI味"、"润色一下"
- 用户说"帮我优化这篇文章"

**不触发：**
- "写文章" → wechat-article-writer
- "审稿" → adversarial-content-review
- "发草稿箱" → wechat-studio

## 不做什么

- 不改文章方向（不重想观点、不换框架）
- 不重写正文（只润色句子，不重构结构）
- 不做选题、不做内容大纲
- 不做发布

## 执行流程

### Step 1: 读取稿件

读取用户指定的 article-draft.md 文件；如果用户直接粘贴正文，就以粘贴正文为稿件来源。提取：
- 标题
- 开头（前 3-4 句）
- 结尾（最后 2-3 句）
- 正文（通读一遍，了解上下文）

### Step 2: 读取定制配置

读取 `../_shared/account-mode-guide.md`、`../_shared/channel-safety-guide.md`、`../_shared/style-guide.md` 和 `../_shared/extend-system.md`，检查是否有：
- 文风样本（`../_shared/style-guide.md`）
- 品牌信息
- 稿件所属账号模式及其可使用的语气和专业身份

### Step 3: 标题优化

读取 `references/packaging-templates.md` 中的标题优化流程。

1. 分析原标题的问题（太平/太长/太模糊/太标题党）
2. 从 8 种标题模板中选最合适的 3 种
3. 生成 3 个候选标题
4. **呈现给用户选择**

### Step 4: 开头优化

读取 `references/packaging-templates.md` 中的开头优化流程。

1. 判断当前开头的类型和问题
2. 从 5 种开头模板中选最合适的 3 种
3. 重写 3 个候选开头
4. **呈现给用户选择**

### Step 5: 结尾优化

读取 `references/packaging-templates.md` 中的结尾优化流程。

1. 判断当前结尾的类型和问题
2. 从 4 种结尾模板中选最合适的 2 种
3. 重写 2 个候选结尾
4. **呈现给用户选择**

### Step 6: 句子润色

读取 `references/packaging-templates.md` 中的句子润色原则和去 AI 痕迹清单。

对正文逐段润色：
- 删冗余词
- 拆长句
- 具体化抽象描述
- 制造节奏变化
- 消除 AI 痕迹
- 删除与账号模式冲突的医生口吻、主播话术或认知导师式断言

润色后的正文**不改变原意和结构**，只优化表达。

### Step 7: 输出

用户确认标题、开头、结尾的选择后，合并到 article-draft.md：
- 覆盖原标题
- 覆盖原开头
- 覆盖原结尾
- 覆盖润色后的正文
- 保留文章预设和主题色字段

输出覆盖原文件：`article-draft-<关键词>.md`

## Guardrails

- 标题/开头/结尾的候选必须呈现给用户选择，不自动替换
- 句子润色不改变原意，只优化表达
- 如果原文没有明显问题，不要为了改而改
- 去 AI 痕迹是必做项，不能跳过
- 优化后的文章字数控制在原字数的 ±10% 以内
