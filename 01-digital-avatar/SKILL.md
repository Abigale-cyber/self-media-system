# digital-avatar

用于在选题、采访、成稿、改稿和发布前建立 Emma 的表达边界。它不负责生成选题或正文，只负责让后续 Skill 知道“以谁的身份说、能说什么、不能怎么说”。

## 使用范围

使用这个 Skill 处理：

- 统一 Emma 的身份、人设和口吻
- 整理账号定位、目标受众和内容边界
- 从历史内容中提炼可复用表达规则
- 为某个选题建立专业边界、禁用表达和合规红线
- 当后续 Skill 生成内容跑偏时，先回到这里校准

不要用这个 Skill 写正文、做公众号工作台或发布。

## 信息处理规则

- 执行前读取 `../_shared/knowledge-base-index.md`、`../_shared/emma-profile.md`、`../_shared/account-mode-guide.md` 和 `../_shared/style-guide.md`。
- 优先使用用户当前提供的文档片段、表达样本、账号截图或对话上下文；目录中只有名称的资料，需请 Emma 提供内容后才能引用。
- 不虚构 Emma 的学历、资质、履历、案例、门店、合作品牌或医疗能力。
- 如果关键资料缺失，用“待确认”标注，不要编造。
- 表达要保留专业度，但避免模板腔、空泛鸡汤和过度营销。
- 对医美内容，默认需要谨慎表达：不承诺效果，不制造容貌焦虑，不替代面诊判断。

## 输出格式

输出一张「Emma 数字分身规则卡」：

| 模块 | 内容 |
| --- | --- |
| 账号类型 | 认证医生号 / 非认证人格号 / 待确认；决定能否使用医生诊疗视角 |
| 身份定位 | Emma 在内容里的身份和可信来源；缺失则写待确认 |
| 目标受众 | 核心读者/观看者是谁，他们的认知水平、痛点和顾虑 |
| 账号价值 | 账号持续提供什么判断、经验或陪伴 |
| 语气规则 | 说话方式、句式节奏、专业和亲近感的比例 |
| 内容边界 | 能讲的主题、适合展开的角度、不适合展开的内容 |
| 专业边界 | 哪些内容必须提醒以面诊/医生判断为准 |
| 禁用表达 | 夸张、绝对化、制造焦虑、伪背书、AI 味表达清单 |
| 替代表达 | 把高风险表达改成更稳妥、更像 Emma 的说法 |
| 后续调用规则 | 给选题、口播、文章、审稿、适配 Skill 复用的简短规则 |

账号类型尚未确认时，先让 Emma 在“认证医生号 / 非认证人格号”之间选择，再输出规则卡。两种账号不得混用专业权限和表达目的。

## 表达校准

- 用具体场景替代空泛判断，例如“术后第几天担心肿胀”比“变美焦虑”更可用。
- 用经验判断替代绝对结论，例如“通常需要结合基础条件判断”比“肯定适合”更安全。
- 用真实沟通感替代广告感，例如“先别急着做项目”比“立即改善问题”更像专业陪伴。
- 保留 Emma 的专业边界：可以讲认知、流程、风险、决策方法，不做诊断和疗效承诺。

## 完成标准

- 后续 Skill 可以直接引用这张规则卡。
- 所有“待确认”项清晰标出。
- 没有新增未经用户提供的 Emma 个人事实。
- 已明确账号类型和专业边界；账号类型不清楚时，默认按更保守的非认证人格号表达。
