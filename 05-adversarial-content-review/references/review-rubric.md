# 审稿评分标准

## 三角色定义

### 笔杆子（内容专家）
关注内容本身的质量：观点是否清晰、论据是否充分、逻辑是否自洽。

**检查重点：**
- 核心观点是否一句话能说清
- 每个章节是否独立交付价值
- 素材/案例是否支撑了观点
- 有没有自相矛盾的地方

### 参谋（读者代言人）
站在目标读者角度：能不能看懂、有没有收获、会不会读完。

**检查重点：**
- 开头 3 句话能不能抓住目标读者
- 有没有信息跳跃（突然出现未解释的概念）
- 节奏是否拖沓（哪里想跳过）
- 读完有没有明确的 takeaway

### 裁判（综合评定）
综合两位意见，做最终裁定。关注全局而非细节。

**检查重点：**
- 标题和内容是否匹配
- 整体结构是否平衡（头重脚轻？虎头蛇尾？）
- 字数是否在合理范围（1500-3000）
- 是否有 AI 痕迹残留

## 五维度评分

| 维度 | 满分 | 0分（差） | 1分（及格） | 2分（好） |
|------|------|----------|-----------|----------|
| 标题吸引力 | 2 | 平淡无感，不想点 | 还行，但不够独特 | 有冲突/悬念/利益点，想点 |
| 结构稳定性 | 2 | 章节之间断裂，跳来跳去 | 能跟上，但有冗余 | 节奏好，每段有价值，读完顺 |
| 论据强度 | 2 | 只有观点没有支撑 | 有案例或专业判断但不够具体 | 有用户素材、专业逻辑或可验证资料支撑 |
| 读者价值 | 2 | 看完没什么可拿走的 | 有点启发但不够清晰 | 读完知道下一步该做什么 |
| 篇幅比例 | 2 | 明显过长或过短 | 基本合适，略有失衡 | 头尾短、中间充实，比例好 |

**总分 = 五项之和（0-10）**

## 审稿结论阈值

| 总分 | 结论 | 处理 |
|------|------|------|
| ≥ 8 | 通过 | 可发布，列出亮点和微调建议 |
| 5-7 | 需修改 | 列出具体修改点，按优先级排序 |
| < 5 | 需重写 | 说明核心问题，建议是否需要回到 content-outline |

## 审稿流程

### 第 1 轮：笔杆子审

逐章节审阅，输出：
- 每个章节的观点清晰度（清晰/模糊/缺失）
- 论据支撑度（充分/不够/空洞）
- 逻辑连贯性（流畅/跳跃/断裂）
- 具体修改建议

### 第 2 轮：参谋审

以目标读者身份通读，输出：
- 哪里想停下来（看不懂）
- 哪里想跳过（不感兴趣）
- 哪里想转发（有共鸣）
- 整体读完感受（一句话）

### 第 3 轮：裁判裁定

综合前两轮意见：
- 五维度打分
- 最终结论（通过/需修改/需重写）
- 如果需修改：按优先级列出修改清单
- 如果需重写：指出核心问题（是方向错了还是执行不行）

## 修改建议格式

```yaml
优先级: P0|P1|P2          # P0=必须改，P1=建议改，P2=可选
位置: string              # 具体到哪个章节/段落
问题: string              # 一句话说清楚问题
建议修改: string          # 具体的改法，不只是"需要优化"
```
